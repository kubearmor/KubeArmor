# KubeArmor v1.6 Release Notes

We are excited to announce the release of **KubeArmor v1.6**, packed with powerful new features, significant enhancements, and critical bug fixes that make workload protection and observability even more robust for cloud-native environments.

This release reflects major advancements in policy enforcement, system monitoring, and ecosystem integrations while addressing important stability and performance improvements.

## v1.6.4 (2025-11-24)

### New Features
- Introduced USB Device Handler for host-level USB observability and enforcement (audit/allow/block). Handles device attach/remove via Netlink uevents and enforces with sysfs authorization. Configurable with `--enableUSBDeviceHandler` and default posture via `--hostDefaultDevicePosture`. (commit 09b6011)

### Improvements
- Added structured `eventData` map to alerts/logs; Feeder now parses key=value pairs from Data/Resource and populates protobuf fields for downstream consumers. (protobuf, feeder)
- Enhanced device policy matching with specificity (class/subclass/protocol/level) and clearer Device log formatting. (feeder)
- RHEL kernel compatibility macro for UDP DNS visibility path in BPF to support RHEL build >= 400. (BPF)
- CI: Added coverage capture for k8s env tests, artifact upload, and timeouts/cleanup tuning in network and UBI test workflows.
- Signing: Switched cosign to bundle signature output (`${artifact}.sigstore.json`) in goreleaser config.

### Bug Fixes
- Controller status check now lists Pods cluster-wide instead of namespace-bound when verifying daemon status. (operator common)

### Breaking Changes
- HostPolicy Device schema changed:
  - `class` remains string but accepts names or numeric codes (decimal/hex); `subClass`, `protocol`, and `level` are now int32 fields; only `class` is required. CRDs updated in deployments/helm and controller APIs. Existing policies using textual `subClass` must be migrated.
- Protobuf API: Alerts and Logs now include `EventData` map; regenerate clients if using compiled protobufs.

### Documentation
- Added USB Device Handler guide with examples and class reference: getting-started/usb_device_handler.md.
- Updated Host Security Policy spec to reflect new device fields and examples; removed legacy usb_device_class.md.

### Dependencies/Tooling
- Protobuf toolchain regenerated with protoc v3.21.12 and updated generated code.

### Upgrade Notes
- Version bumped to v1.6.4 (STABLE-RELEASE). If you use HostPolicy device rules, update policies to numeric `subClass`/`protocol` where applicable and verify CRDs are refreshed.
- If consuming logs/alerts downstream, adapt to `EventData` map and updated USB Device log `Resource` format.
- To enable USB enforcement, set `enableUSBDeviceHandler=true` (and ensure HostPolicy is enabled); optionally set `hostDefaultDevicePosture` to `audit` or `block`.

## Key Features & Enhancements

### üîê **Advanced Process Arguments Matching**

- Introduced **argument-based matching for processes** in policies.

- Allows precise control over command-line arguments, enabling granular process enforcement.

- This feature is currently limited to BPFLSM.

- **Example policy:**

  ```yaml
  apiVersion: security.kubearmor.com/v1
  kind: KubeArmorPolicy
  metadata:
    name: allow-steampipe-args
  spec:
    selector:
      matchLabels:
        app: steampipe
    process:
      matchPaths:
      - path: /usr/bin/python3.6
        allowedArgs:
          - -m
          - modules.steampipe_aws
    action: Block
  ```

### üì° **DNS Visibility at Pod-Level**

- Added **DNS query tracing** on UDP to provide insights into domain lookups from workloads.
- Helps detect malicious behaviors like **DGA (Domain Generation Algorithms)** or unauthorized C2 communications.

### üõ°Ô∏è **New Policy Presets**

- **ProtectProc:** Blocks unauthorized access to the `/proc` directory by non-owner processes.
- **ProtectEnv:** Prevents unauthorized access to sensitive environment variables in `/proc/[pid]/environ`.
- **ExecPreset:** Enforces restrictions on external process executions (e.g., via `kubectl exec`).

### üîå **Container Runtime Enhancements**

- **OCI Hooks Support:**
  - Added support for **containerd** and **CRI-O hooks**, eliminating the need for exposing runtime UNIX sockets for container events.

### üìà **Improved Telemetry and Observability**

- Added **TTY information** in BPF-LSM generated telemetry.
- Enhanced telemetry with **network metadata** using Kubernetes informers.
- Extended alert resources to include **full command arguments**.

### üåê **Ecosystem and Integrations**

- **OpenSearch Support:** Added OpenSearch as a datasource for process graphs in Grafana dashboards.
- Integrated **image vulnerability scanning workflows** (via Trivy) in release pipelines.
