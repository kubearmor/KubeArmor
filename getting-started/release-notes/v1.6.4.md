# KubeArmor v1.6.4 Release Notes

Release date: 2025-11-05

## New Features

- Added USB Device Observability to surface host USB device events and metadata in telemetry. Updated CRDs and monitoring pipeline to expose vendor/product/class information. (PR #2172)
- Added USB Device Enforcement with a new USB Device Handler to allow/deny device classes and attributes. Enables policy-based control over USB usage on hosts. (PR #2194)
- Added File Integrity Monitoring (FIM) using Linux IMA hash matching, including dynamic enable/disable of the IMA module and improved IMA visibility in the system monitor. (PR #2170)
- Added EventData field in telemetry to deliver structured event payloads for easier downstream parsing and analytics. (PR #2204)
- Added support for unorchestrated Podman via an OCI hook (rootful and rootless), enabling enforcement and visibility outside Kubernetes clusters. (PR #1874)

## Improvements

- Operator: Added KubeArmorConfig support to configure nodeSelector and environment variables for operator-managed components. (PR #2187)
- Helm: Corrected tolerations indentation in the operator chart template for reliable scheduling. (PR #2138)
- Operator: Moved imagePullSecrets to .Values.kubearmorOperator.image.imagePullSecrets for consistency. (PR #2190)
- Compatibility: Added docker.io prefix to image repositories for CRI-O 1.34 compatibility. (commit c8e52045)

## Bug Fixes

- Fixed missing action field in syscall logs. (PR #2169)
- Fixed preset alerts to correctly populate the action field. (PR #2210)
- Fixed panic when updating controller nodeSelector. (PR #2193)
- Fixed missing labels in emitted events. (commit b54a8d93)
- Fixed system monitor compilation issues on RHEL kernels. (commit 5251c9cb)

## Dependencies

- Bumped cilium/ebpf to resolve BTF parsing issues. (PR #2200)
- Upgraded Go toolchain and Docker package; addressed gosec/govuln findings. (commits 79083909, 94386b37, 916429fa, 306a2814)

## Documentation

- Updated FAQ with instructions to enable BPF-LSM on RHEL using grubby. (PR #2174)
- Fixed test script paths in the testing guide. (PR #2144)

## Known Issues

- None noted in this release.
