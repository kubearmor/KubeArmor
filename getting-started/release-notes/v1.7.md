# KubeArmor v1.7.x Release Notes Summary

## Version 1.7.4

### ✨ New Features & Enhancements
- **Policy Matching:** Improved policy matching for hostnames to ensure more accurate host-based enforcement.
- **Healthchecks:** Added a new gRPC-based healthcheck for liveness probes, improving daemon stability monitoring.
- **DNS Enforcement:** Added Top-Level Domain (TLD) and subdomain enforcement.
- **Security & Dependencies:**
  - Upgraded Go versions to address known Go vulnerabilities (`go-vuln`).
  - Updated CI Actions: `actions/checkout` to v3.6.0, `trivy-action` to v0.36.0, and updated the `podman` module to v6.0.0.

### 🐛 Bug Fixes
- **Containerd Events:** Fixed an issue by using the `TaskExit` type to properly unmarshal containerd exit events.
- **Operator Permissions:** Added `NET_ADMIN` to the Security Context Constraints (SCC) to ensure the operator has the necessary networking privileges.
- **Daemon Crashes:** Fixed a critical panic caused by a `send on closed channel` error.
- **UBI Images:** Updated the Snitch UBI (Universal Base Image) to run as non-root.
- **CI/CD:** Fixed the CI pipeline to properly certify images on Red Hat ecosystems.

---

## Version 1.7.3

### ✨ New Features & Enhancements
- **Policy Watchers:** Migrated from standard watchers to using Kubernetes informers to watch Host and Network policies, significantly improving performance and reducing API server load.

### 🐛 Bug Fixes
- **System Monitor:** Fixed a BPF verifier error in the system monitor that occurred specifically on Ubuntu 18.04 using Clang 12.
- **CI/CD:** Fixed the `ci-test-systemd` workflow by explicitly installing the `protoc` binary.

---

## Version 1.7.2

### ✨ New Features & Enhancements
- **GKE Support:** Added the `gkeAllowlistVersion` flag to improve compatibility and control over GKE deployments.
- **Security & Dependencies:**
  - Upgraded core Go dependencies to resolve security vulnerabilities.
  - Bumped the `podman` package to fix a known vulnerability.
  - Updated the frontend `jquery` dependency to v3.7.1.

### 🐛 Bug Fixes
- **CPU Architecture Compatibility:** Fixed a critical issue where the init container was incompatible with older CPUs that do not support the `x86-64-v3` microarchitecture instruction set.
- **Controller Scheduling:** Fixed an issue with controller toleration updates.
- **Container Builds:** Fixed UBI init container build issues and updated the KubeArmor Dockerfile to safely handle package updates.
- **Documentation:** Corrected spelling mistakes across the entire project repository.

---

## Version 1.7.1

### ✨ New Features & Enhancements
- **UBI Build Pipeline:** Separated the build pipelines for Universal Base Images (UBI). The operator, controller, snitch, kubearmor daemon, and kubearmor-init images are now built completely independently.
- **Security & Dependencies:**
  - Bumped all `go.mod` files uniformly to Go 1.26.3.
  - Updated various Go dependencies to patch known security vulnerabilities.

### 🐛 Bug Fixes
- **UBI Releases:** Fixed a bug related to the automated release of controller and operator UBI images.

---

## Version 1.7.0

### ✨ New Features & Enhancements
- **DNS Enforcement Core:** Introduced brand-new **DNS Enforcement** capabilities directly into KubeArmor.
- **Audit Support:** Added support for `Audit` actions in DNS and network rules, allowing users to observe traffic without blocking it.
- **Testing:** Added comprehensive DNS network tests to the testing suite.

### 🐛 Bug Fixes
- **Allow Policies:** Fixed several bugs related to `Allow` actions in network policies.
- **DNS Rules:** Fixed specific rule generation bugs affecting DNS network rules.
